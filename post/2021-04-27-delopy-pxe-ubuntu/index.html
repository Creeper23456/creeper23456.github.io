部署Ubuntu系统并通过PXE进行无盘启动

2021-04-27 Views2008字9 min read繁

前言

在业余时间部署客户需要的应用，发现服务器不够用了，看了一眼早已被遗弃的笔电，我眼中一亮，准备发挥我的传统艺能：废物利用

准备

在开始之前，你需要准备好：

  • 一台支持PXE的设备，一般来说只要不是太过时，基本上都支持。
  • 一台支持TFTP功能的DHCP服务器。
  • pxelinux固件
  • 局域网内能正常运行的Debian系服务器（包括Ubuntu）

开始部署

启用群晖iSCSI功能并挂载iSCSI远程磁盘

iSCSI起源于2003年的IBM公司，通过TCP/IP协议模拟SCSI总线以方便商用服务器的弹性存储部署，要想在群晖NAS上启用这一功能，首先登录NAS并打开存储空间管理员程序：

可以看到我的iSCSI设备已经正常运行并且已经有设备挂载了，为了教学，我们切换到iSCSI Target标签，创建一个iSCSI目标：

接下来创建iSCSI LUN，为新的iSCSI连接点设置设备，对于群晖来说，这里创建了一个虚拟磁盘供其他设备使用：

LUN名称可以随意填写，容量我设置为50GB，在iSCSI Target下拉菜单处选择刚刚创建的Target。

我们回到VPS的SSH窗口，首先安装iSCSI客户端程序：

apt install open-iscsi -y

安装完成之后，编辑文件：

vim /etc/iscsi/iscsid.conf

在大约第43行附近改为：

node.startup = automatic

设置好后iSCSI服务会伴随系统启动，然后开始搜寻iSCSI设备：

iscsiadm --mode discovery --type sendtargets --portal [Your NAS's IP addr]

可以看到系统已经能认出远程硬盘了，但还需要分区并格式化才能使用：

fdisk /dev/sdb
mkfs.ext4 /dev/sdb

完成之后还需要保证iSCSI在重启后能够被系统自动挂载，我是这样做的：

mkdir /mnt/iscsi #创建挂载点
tune2fs -l /dev/sdb #查看iSCSI设备的UUID
vi /etc/fstab #写入fstab

其中fstab文件添加下列内容：

UUID=[Your disk's UUID] /mnt/iscsi ext4 _netdev 0 0

重启，大功告成！

创建系统文件

首先进入/mnt/iscsi，这是接下来的工作目录。

安装必要的软件：

apt install schroot debootstrap -y

之后创建chroot容器必要的文件：

debootstrap bionic /mnt/iscsi http://archive.ubuntu.com/ubuntu/

此命令会创建一个基于Ubuntu server 18.06的chroot系统，之后我们的无盘机也会以该文件系统为根目录启动。

如果下载速度慢，尝试使用代理下载。

完成之后，输入命令来到chroot容器：

chroot /mnt/iscsi

在操作前，切换到TUNA软件源：

#向/etc/apt/sources.list文件内写入
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu bionic main restricted
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu bionic universe
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu bionic multiverse
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu bionic-update main restricted
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu bionic-update universe
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu bionic-update multiverse
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu bionic-security main restricted
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu bionic-security universe
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu bionic-security multiverse

执行下面的指令简易初始化接下来要用的系统：

apt update
apt install openssh-server -y #安装SSH服务器，记得允许root身份登录
passwd root #设置新的root密码
hostnamectl set-hostname [Your new hostname] #设置一个新的主机名，不然无盘机启动后主机名与VPS冲突

创建启动需要的内核，在之前请确认软件源设置无误。

apt install linux-image-generic

安装时可能遇到这样的界面，此处选YES，我们不使用GRUB引导操作系统。

安装完成后，执行：

update-initramfs

输入exit退出chroot容器，将生成的initrd.img文件放好备用

至此，文件系统创建完成。

准备NFS服务器

安装NFS服务器：

apt install nfs-kernel-server -y

之后修改配置文件：

vi /etc/exports

在文件内添加一行：

/mnt/iscsi *(rw,sync,no_root_squash)
#格式为 /path/to/your/root [ipaddr](rw,sync)，可自行修改

重启NFS服务器并设置开机自启：

systemctl restart nfs-kernel-server.service
systemctl enable nfs-kernel-server.service

准备DHCP服务器

我使用的是基于嵌入式Linux的Pandavan系统路由器，dnsmasq已经集成了TFTP功能，在配置文件处启用即可。

将刚刚生成的内核文件复制到TFTP服务器的虚拟根目录。

将PXE固件上传至路由器，并修改pxelinux.cfg/default文件

DISPLAY boot.txt2 #此处为帮助信息，可写可不写，txt文件放置于TFTP根目录处
...

DEFAULT linux #默认启动菜单

LABEL linux #启动菜单
    kernel vmlinuz
    append vga=normal initrd=initrd.img ramdisk_size=14332 root=/dev/nfs nfsroot=[;Your NFS server's ip addr]:/mnt/iscsi rw -- #此处为引导信息，尤其是nfsroot后的参数请修改为自己的NFS服务器

PROMPT 0
TIIMEOUT 0 #超时零秒，会自动启动默认启动

保存设置，使用虚拟机测试，无问题后启动实体机，大功告成

结语

这样一个简单的网络启动折腾了自己很久，网络上很多教程都需要自行编译Linux内核，这篇教程是自己按着ipxe的一些文档以及askubuntu社区的部分问答而成的

因为网络启动Linux这一需求较为特殊，国内更多是部署只读无盘然后用来Centos自动化安装的，因此写了这样一篇教程，希望能帮到各位。
