iPod touch 4变身Web服务器 – 越狱之后的环境部署

2021-01-20 Views601字3 min read繁

回顾

上一篇博文详细描述了iPod4越狱的步骤及注意事项。问题也很明显，每次iPod关机/重启之后越狱便失效了，甚至App Store安装的正版程序也闪退，所幸Cydia有补丁可以解决该问题，我们搜索并安装即可。

保持越狱

在设备已经被电脑越狱工具引导启动后Cydia可用的情况下，启动Cydia。

如果是第一次启动Cydia，请务必等待屏幕顶上提示的更新完成，Cydia是和Debian同款的apt包管理器的iOS图形界面，而屏幕顶上的更新实为更新软件源。

源更新完成后，在Cydia内搜索“p0sixspwn”并安装。

重启后你会发现，Cydia依旧能正常运行，不再需要电脑进行引导启动了。

配置系统

既然是建站，那就得以我们熟悉的方式搭建和维护咯。

在Cydia上安装apt和OpenSSH这两个工具之后，就可以开始折腾这台小小的iPod，首先是安装一些常用的工具：

apt-get install top wget whois vim shell-cmds curl nano -y --force-yes

注意：Cydia的一些源过不了验证，所以要加force-yes参数

这儿遇到了一个坑，如果你的终端提示：

iPod-touch-4:~ root# apt-get update
E: Could not get lock /var/lib/apt/lists/lock - open (35: Resource temporarily unavailable)
E: Unable to lock the list directory

请关闭前台正在运行的Cydia，然后继续。

修改默认密码：

passwd root

初次登录SSH可使用默认密码“alpine”，但安全起见最好修改默认密码。

之后打开Cydia，添加下面的源：

https://cydia.akemi.ai/webstack/

apt update一下，然后执行安装：

apt-get install ios-lighttpd-php-mysql

之后配置站点的步骤与配置普通的lighttpd站点相同。设置好反向代理然后穿透到互联网即可。

注意事项

  • 对于CC攻击，经过ab压测，在前端Web服务器无WAF的情况下会出现504错误，此时iOS本身不受影响，貌似iOS自身的进程优先级大于命令行进程。
  • 目前博主并不是很清楚如何重载一个lighttpd服务器，php可通过修改配置文件实现多版本共存，php是以fcgi模式工作的。
  • iPod touch 4是32位arm设备，在自行编译软件包时请注意体系架构。
